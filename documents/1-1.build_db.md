# 1-1. PostgreSQL 로컬 DB 구축

## 설치 정보

| 항목 | 값 |
|------|-----|
| DBMS | PostgreSQL 17.9 |
| OS | Ubuntu 24.04.4 LTS |
| 설치 방식 | APT (공식 PostgreSQL 저장소) |
| 포트 | 5432 (기본값) |
| 인코딩 | UTF-8 |
| 로케일 | ko_KR.UTF-8 |
| 설정 파일 | `/etc/postgresql/17/main/postgresql.conf` |
| HBA 파일 | `/etc/postgresql/17/main/pg_hba.conf` |
| 데이터 디렉토리 | `/var/lib/postgresql/17/main` |

## 계정 정보

| 항목 | 값 |
|------|-----|
| 사용자명 | `glovis` |
| 비밀번호 | `Glovis1!` |
| 권한 | SUPERUSER, CREATEDB, CREATEROLE |
| 소유 DB | `gapttuk` |

## 접속 방법

```bash
# CLI 접속
PGPASSWORD='Glovis1!' psql -U glovis -d gapttuk -h localhost

# 또는 Connection String
postgresql://glovis:Glovis1!@localhost:5432/gapttuk
```

## 서비스 관리

```bash
# 시작
sudo systemctl start postgresql

# 중지
sudo systemctl stop postgresql

# 재시작
sudo systemctl restart postgresql

# 상태 확인
sudo pg_isready
```

## 다음 단계

### 1. 스키마 설계
값뚝 앱에 필요한 테이블을 설계해야 합니다. 주요 테이블 후보:

- **users** - 사용자 정보 (인증, 프로필)
- **products** - 추적 대상 상품 정보
- **price_alerts** - 가격 알림 설정 (목표 가격, 알림 조건)
- **price_history** - 상품 가격 변동 이력
- **subscriptions** - 유료 구독 정보
- **notifications** - 발송된 알림 기록

### 2. 마이그레이션 도구 선택
스키마 버전 관리를 위한 마이그레이션 도구를 정해야 합니다:

- **Prisma** - TypeScript ORM + 마이그레이션 (가장 인기)
- **Drizzle** - 경량 TypeScript ORM
- **Knex.js** - SQL 쿼리 빌더 + 마이그레이션
- **raw SQL** - 직접 마이그레이션 스크립트 관리

### 3. 앱 연동
선택한 프레임워크(React Native 등)에서 DB 연결 설정:

```
DATABASE_URL=postgresql://glovis:Glovis1!@localhost:5432/gapttuk
```

`.env` 파일에 위 값을 설정하고, 선택한 ORM/쿼리빌더로 연결합니다.
