### 2.2 시나리오 1: 상품 검색 → 가격 확인 → 알림 설정

```
사용자 의도: "에어팟 가격이 떨어지면 알려줘"

  [HOME]
    │
    │  상단 검색바 탭
    ▼
  [SEARCH]
    │  "에어팟" 입력  ← 인기검색어 참고 가능
    │  (또는 쿠팡 URL 붙여넣기)
    ▼
  [SEARCH_RESULT]
    │  필터: 역대최저 / 하락추세
    │  정렬: 할인률순
    │  상품 카드 탭  ← "23일만에 최저가" 배지 확인
    ▼
  [PRODUCT_DETAIL]
    │  ┌─ 가격 그래프 확인 (시계열 + 요일별)
    │  ├─ 가격하락확률 게이지 확인 (예: 94% "지금사야해")
    │  ├─ 최저가↔최고가 슬라이더로 현재 위치 파악
    │  └─ "알림 설정" 버튼 탭
    ▼
  [ALERT_SETTING]  (바텀시트 또는 인라인)
    │  4단계 프리셋 중 선택:
    │    ○ 지정가 이하 (직접 입력)
    │    ○ 평균가 이하
    │    ○ 최저가 근접
    │    ○ 역대 최저가
    │  "알림 받기" 버튼 탭
    ▼
  [완료]  ← 즐겨찾기에 자동 추가 + 확인 토스트
```

### 2.3 시나리오 2: 알림 수신 → 상품 확인 → 쿠팡 구매

```
사용자 의도: "알림 왔네, 싸졌으면 바로 살래"

  [푸시 알림 / 웹 알림]
    │  "에어팟 Pro 2 — 역대최저가 도달! 219,000원 (-15%)"
    │  알림 탭
    ▼
  [PRODUCT_DETAIL]
    │  ┌─ 현재가 확인: 219,000원
    │  ├─ 가격하락확률 게이지: "지금사야해" (97%)
    │  ├─ "35,000원 싸게 구매하기" CTA 확인
    │  └─ 하단 고정 CTA "쿠팡에서 구매하기" 탭
    ▼
  [쿠팡 외부 이동]  ← 쿠팡파트너스 링크 경유
    │  구매 완료
    ▼
  [커미션 발생]  ← 24시간 내 구매 시 ~3% 수익
```

### 2.4 시나리오 3: 카테고리 패시브 알림 → 자동 추천 → 구매

```
사용자 의도: "가전/디지털 카테고리 싼 거 있으면 알아서 알려줘"

  [MY_PAGE]
    │  "카테고리 알림" 메뉴 탭
    ▼
  [CATEGORY_ALERT]
    │  "+" 카테고리 추가하기 탭
    │  카테고리: "가전/디지털" 선택
    │  조건: "20% 이상 하락 시" 선택
    │  최대 가격: "500,000원" 입력
    │  "알림 등록" 탭
    ▼
  [완료]  ← "등록 완료! 조건에 맞는 상품 발견 시 알림을 보내드립니다"

  ... (수일 후) ...

  [푸시 알림]
    │  "🔥 가전/디지털 핫딜! LG 스탠바이미 27% 하락 — 489,000원"
    │  알림 탭
    ▼
  [PRODUCT_DETAIL]
    │  ┌─ AI 예측: "지금사야해" (89%)

    ▼
  [쿠팡 외부 이동]
```

```

    ▼
  [EVENT]
    │  "오늘의 가격 퀴즈" 탭

    ▼
  [REWARDS]
    │  현재 보상 잔액 확인
    │  기프티콘 교환 탭
    ▼
  [완료]  ← "기프티콘이 발급되었습니다!"
```

## 3. 각 화면별 정보 구조 (와이어프레임)

### 3.1 메인 홈 (HOME)

**설계 근거**: 로우차트의 필터/정렬 + 지니알림의 인기검색어 + 역대가의 카테고리 태그 + 출석/이벤트 진입점 통합

**변경점 (v0.1 → v0.2):**
- 인기검색어 자동 스크롤링 영역 강화 (뉴스 티커 스타일)
- 출석체크 + 이벤트 퀵 진입 배너 추가
- 포인트 잔액 표시 (헤더)

## 4. 네비게이션 구조

### 4.1 하단 탭바 (앱)

```
┌────────────────────────────────────────────────────────────┐
│  🏠 홈    🔍 검색    ⭐ 즐겨찾기    🔔 알림    👤 마이    │
│  Home    Search   Favorites   Alerts    My      │
└────────────────────────────────────────────────────────────┘
```

| 탭 | 아이콘 | 이동 화면 | 배지 |
|---|---|---|---|
| 홈 | 집 | `HOME` | 없음 |
| 검색 | 돋보기 | `SEARCH` | 없음 |
| 즐겨찾기 | 별 | `FAVORITES` | 즐겨찾기 수 |
| 알림 | 종 | `ALERT_CENTER` | 미읽은 알림 수 (빨간 점) |
| 마이 | 사람 | `MY_PAGE` | 없음 |

**탭 5개 선정 근거:**
- 홈/검색/마이 : 필수 3탭 (모든 경쟁사 공통)
- 즐겨찾기 : 폴센트/역대가에서 핵심 기능으로 확인 (관심 상품 빠른 접근)
- 알림 : 가격 알림이 앱의 핵심 가치이므로 탭 레벨로 승격 (지니알림의 벨 아이콘 참고)

### 4.2 상단 헤더 구성

**앱 헤더 (홈)**
```
┌────────────────────────────────────────────────────────────┐
│  값뚝                             [🔔 3]  [👤]       │
├────────────────────────────────────────────────────────────┤
│  🔍 검색바 (탭 시 검색 화면으로 이동)                      │
└────────────────────────────────────────────────────────────┘
```

**앱 헤더 (상세 페이지)**
```
┌────────────────────────────────────────────────────────────┐
│  ← 뒤로      [페이지 타이틀]       [공유] [♡]             │
└────────────────────────────────────────────────────────────┘
```

**웹 헤더 (GNB)**
```
┌─────────────────────────────────────────────────────────────────────┐
│  [로고] 값뚝   🔍 [검색바 (넓게)]   [로그인] [회원가입]       │
│                                                                     │
│  홈 | 카테고리 ▼ | 핫딜 | 키워드알림 | 즐겨찾기 | 알림센터 | 마이   │
└─────────────────────────────────────────────────────────────────────┘
```

### 4.3 웹 vs 앱 UI 차이점

| 요소 | 앱 | 웹 |
|---|---|---|
| **네비게이션** | 하단 탭바 (5탭) | 상단 GNB (수평 메뉴) |
| **카테고리** | 가로 스크롤 태그 바 | 좌측 고정 사이드바 (로우차트 방식) |
| **상품 그리드** | 2열 | 3~4열 (반응형) |
| **검색** | 전체화면 전환 | 헤더 내 인라인 |
| **알림 설정** | 바텀시트 모달 | 인라인 폼 (페이지 내) |
| **가격 그래프** | 세로 스크롤 (풀폭) | 가로 넓게 (더 많은 데이터 포인트) |
| **하단 CTA** | 고정 하단바 | 우측 사이드 고정 또는 스크롤 따라감 |
| **기능 제한** | 없음 (풀기능) | **없음 (풀기능)** ← 폴센트와 핵심 차별점 |
| **알림 방식** | 푸시 알림 | 웹 푸시 + 이메일 (선택) |
| **앱 유도** | 해당없음 | 부드러운 유도 (팝업 아님, 배너만) |

**핵심 차별점: 웹에서도 풀기능 제공**
> 근거: 폴센트는 웹에서 검색 제한, 그래프 가림, QR코드 고정 등 공격적 앱 유도 전략을 사용함.
> 로우차트는 웹에서 알림 설정까지 가능하여 사용자 경험이 우수함.
> 값뚝는 로우차트 방식을 따르되, 웹 푸시 알림까지 지원하여 앱 설치 없이도 핵심 가치를 제공.
