🔷 STEP 0. 사전 준비
• PRD 1페이지 작성 (목표 + 기능 + 사용자)
• AI에게 README 생성 (요구사항 + 기술스택 + 마일스톤 ≤5)
• 기술스택 최대한 단순하게
• AI IDE 글로벌 룰 설정
• AI 모델의 라이브러리 버전 확인

🔷 STEP 1. 코드리서치 [방어: 무지한 변경 차단]
• "이 폴더를 깊이 읽고 research.md 작성. 아직 구현하지 마"
• 디렉토리 구조, 데이터 흐름, 외부 의존성, 기존 패턴 파악

🔷 STEP 2. 계획 [방어: 잘못된 변경 차단]

반복 루프:

2-1. AI가 plan.md 초안 작성
     (접근방식 + 코드 스니펫 + 파일경로 + 트레이드오프 + 마일스톤)
2-2. 인간이 에디터에서 검토
2-3. plan.md에 인라인 메모 추가 (4가지 유형)
     - 가정 수정 / 접근 거부 / 제약 추가 / 도메인 지식
2-4. "메모 반영해서 업데이트해. 아직 구현하지 마"
2-5. AI가 plan.md 업데이트 → 2-2로 (만족할 때까지)

plan.md에 수정 금지 섹션 포함:

## ⛔️ DO NOT TOUCH
- [절대 수정 불가 파일/모듈 목록]

🔷 STEP 3. 주석달기 [방어: 내 판단 주입]
• "plan.md 마일스톤 1에 주석으로 로직만 작성. 코드 쓰지 마"
• 검토 → 피드백 → 반복
• 주석이 완벽할 때까지

🔷 STEP 4. 구현 [방어 완료 → 깨끗한 실행]
• 구현 프롬프트:

전부 구현해라.
- 완료 시 plan.md에서 ☑️ 표시
- 모든 단계 끝날 때까지 멈추지 마
- any/unknown 타입 금지
- typecheck 지속 실행, 새 에러 만들지 마
- 기존 테스트 깨지면 즉시 수정

• 매 마일스톤 Git 커밋

🔷 STEP 4.5. 테스트
• 단위/통합 테스트 작성 및 실행

🔷 STEP 5. 피드백 & 방향 조정

방향 조정 4원칙:
① 좋은 건 취하고 나머진 버리기 (cherry-pick)
② 과감하게 쳐내기 (불필요한 추상화 제거)
③ 건드리면 안 되는 선 긋기 (수정 금지 영역)
④ 기술 선택은 내가 하기 (라이브러리/패턴 지정)

롤백 패턴:

"전부 이전 커밋 되돌려"
"이제 [딱 하나만] 해. 그것 외엔 아무것도 하지 마"

3번 실패 → Git 롤백 → 범위 축소 재시도
대화 너무 길면 → 새 대화 + research.md/plan.md (http://research.md/plan.md) 컨텍스트

🔷 STEP 6. 보안/문서 리뷰
• API 키 노출, 입력값 미검증, 의존성 취약점 체크
• CHANGELOG.md 업데이트

🔑 황금 규칙 7가지

1. "아직 구현하지 마" — 리서치/계획/주석 단계마다 명시
2. Shared Mutable State — plan.md에 인라인 메모로 소통
3. 한 번에 하나만 — 마일스톤·파일 단위로 진행
4. Git 커밋 자주 — 롤백이 생명줄
5. 방향 조정은 인간이 — 기술 선택, 경계선, 쳐내기
6. any/unknown 금지 — AI의 타입 편법 차단
7. 멈추지 마 + 완료 표시 — 계획 확정 후엔 끝까지 달려
